# Open Data Plan

The project depends on open data to make AI public-service decisions explainable and repeatable.

## Initial Data Categories

| Category | Example Use | Needed Fields |
| --- | --- | --- |
| Public-service directories | Care navigation, benefit routing | service name, eligibility, channel, location, language, accessibility |
| Transportation feeds | Care access, disaster evacuation | route, disruption, accessibility, service hours |
| Weather and hazard alerts | Disaster support | alert type, location, severity, validity window, source |
| Shelter and resource capacity | Disaster support | capacity, accessibility, supplies, freshness |
| School access indicators | Education support | broadband, device access, support services |
| Accessibility metadata | All domains | screen-reader readiness, wheelchair access, language support |

## Recommended Improvements

- Publish source freshness and responsible-agency metadata.
- Add multilingual service descriptions.
- Add accessibility fields as first-class structured data.
- Provide stable identifiers for public services and validation fields.
- Release anonymized aggregate outcome data for bias monitoring.

## Validation Approach

1. Start with open datasets and public-service directories.
2. Add partner-provided field data after privacy review.
3. Compare service recommendations and AI outputs across inclusion personas.
4. Publish aggregate validation results and data-quality gaps.
